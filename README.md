# Задача "Понимание JVM"

## Описание
Проанализировать код ниже и описать каждую строку с точки зрения происходящего в JVM

## Код для исследования
```java
public class JvmComprehension {
    public static void main(String[] args) {
        int i = 1;                      // 1
        Object o = new Object();        // 2
        Integer ii = 2;                 // 3
        printAll(o, i, ii);             // 4
        System.out.println("finished"); // 7
    }
    private static void printAll(Object o, int i, Integer ii) {
        Integer uselessVar = 700;                   // 5
        System.out.println(o.toString() + i + ii);  // 6
    }
}
```

### Строка 1
Загрузчик классов (ClassLoader) загружает в область памяти Metaspace классы:
- класс `JvmComprehension`
- класс `Object`
- класс `Integer`
- класс `System`
- класс `String`
- а также системные классы, необходимы для работы компилятора и JVM

### Строка 2
В области памяти Stack Memory создается Frame для метода `main()`

### Строка 3
Во фрейме метода `main()` создается переменная премитивного типа `i`

### Строка 4
Во фрейме метода `main()` создается переменная ссылочого типа `о` класса `Object`, которая хранит в себе ссылку на объект.  
В области памяти Heap (куча) создается объект класса `Object` (ссылка на него хранится в переменной о)

### Строка 5
Во фрейме метода `main()` создается переменная ссылочого типа `ii` класса `Integer`, которая хранит в себе ссылку на объект.  
В области памяти Heap (куча) создается объект класса `Integer` (ссылка на него хранится в переменной `ii`)

### Строка 6
В Stack Memory создается новый фрейм для метода `printAll()`. Для этого метода берутся аругменты, которые уже есть в памяти, а именно из фрейма метода `main()`.  
Значение `i` берется прям из переменной, так это примитивный тип. Значения объектов `ii` и о берутся по ссылкам (сами объекты остаются хранится в куче).

### Строка, с комментарием  `//5`
На этом этапе во фрейме метода `printAll()` создается переменная `uselessVar` класса `Integer`, которая содержит в себе ссылку на объект, созданный в куче.  
Значение объекта `uselessVar` равно 700 и это информация хранится также в куче.

### Строка, с комментарием  `//6`
Создается фрейм для метода `println()`, в который передаются переменные `о`, `i` и `ii`.  
Создается фрейм для метода `toString()` для объекта `o`

### Строка 7
Создается фрейм для метода `println()`, в который передаются строка "finished".

### Последний шаг
Запускается сборщик мусора, который удаляет из памяти неиспользуемые объекты:
- объекты, на которые нет ссылок
- объекты, которые не используются в ходе выполения программы, даже если на них есть ссылки
